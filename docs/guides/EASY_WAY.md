# 🚀 Самый простой способ отправить запрос

## ❌ Проблема с Swagger UI

При вставке JSON вручную в Swagger UI возникают ошибки с экранированием символов.

## ✅ Решение: Используйте готовый скрипт!

**Не нужно вручную заполнять Swagger UI!** Используйте готовый скрипт:

```bash
python scripts/cli/send_review_request.py
```

Этот скрипт:
- ✅ Автоматически читает файлы из `C:\review`
- ✅ Правильно экранирует все символы
- ✅ Отправляет запрос без ошибок
- ✅ Показывает Task ID для проверки результатов

## 📋 Что делает скрипт

1. Читает все `.md` и `.txt` файлы из `C:\review`
2. Объединяет их в один документ
3. Отправляет запрос на анализ
4. Показывает Task ID

## 🎯 После запуска скрипта

Вы получите Task ID, например:
```
Task ID: ededa7bd-d7f4-4a1f-8c28-05a8bb496396
```

Затем можете:
1. Проверить статус в Swagger UI: `GET /api/v1/review/{task_id}/status`
2. Получить результаты: `GET /api/v1/review/{task_id}/results`
3. Или использовать скрипт `review_from_folder.py` для полного цикла

## 💡 Альтернатива: Полный цикл

Если хотите автоматически дождаться результатов:

```bash
python scripts/cli/review_from_folder.py
```

Этот скрипт:
- Отправит запрос
- Будет ждать завершения
- Покажет результаты
- Сохранит отчет в `C:\review\review_report.md`

## ✅ Рекомендация

**Используйте скрипты вместо ручного заполнения Swagger UI!**

Это проще, быстрее и без ошибок. 🎉
