# 📝 Как заполнить поле "document" в Swagger UI

## 🎯 Главное правило

**Вставьте содержимое вашего файла в поле `document`.**

---

## ✅ Самый простой способ (3 шага)

### Шаг 1: Откройте файл

Откройте ваш файл с документацией в текстовом редакторе (Блокнот, VS Code, и т.д.)

### Шаг 2: Скопируйте текст

1. Выделите весь текст (Ctrl+A)
2. Скопируйте (Ctrl+C)

### Шаг 3: Вставьте в Swagger UI

1. Откройте http://localhost:8000/docs
2. Найдите `POST /api/v1/review/start`
3. Нажмите "Try it out"
4. В поле `document` вставьте скопированный текст (Ctrl+V)
5. В поле `document_type` укажите: `markdown`
6. В поле `context` оставьте: `{}`
7. Нажмите "Execute"

**Готово!** ✅

---

## 📋 Что именно копировать?

Копируйте **весь текст** из вашего файла, включая:
- Заголовки (например, `# Заголовок`)
- Разделители (`---`)
- Весь текст документации
- Все символы и форматирование

---

## 🔍 Проверка правильности

После вставки в поле `document` проверьте:

- ✅ Текст начинается с содержимого файла
- ✅ Текст содержит вашу документацию
- ✅ Текст заканчивается содержимым документации
- ✅ В тексте есть все разделы из вашего файла

---

## ❓ Частые вопросы

### Вопрос: Нужно ли открывать файл вручную?

**Ответ:** ДА! Вы сами открываете файл, копируете его содержимое и вставляете в Swagger UI.

### Вопрос: Что если несколько файлов?

**Ответ:** Объедините содержимое всех файлов в один текст и вставьте в поле `document`.

### Вопрос: Можно ли скопировать только часть текста?

**Ответ:** Лучше скопировать весь текст - так система проанализирует всю документацию.

### Вопрос: Что если файл не в формате .md или .txt?

**Ответ:** Откройте файл в текстовом редакторе и скопируйте текст. Если это бинарный формат (например, .docx), сначала конвертируйте в текст.

---

```
1. Откройте файл в текстовом редакторе

4. Укажите "markdown" в document_type

```
